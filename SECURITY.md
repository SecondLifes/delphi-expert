## Reporting a Vulnerability

We take security seriously. If you find a security vulnerability in this kit — a rule, skill, or generator script that could lead to unsafe behavior, credential exposure, or unintended file/data loss — please report it via **[FILL IN: repo URL]/issues** using the `security` label, or privately at **baspinar99@gmail.com** if the issue shouldn't be public yet.

Please include:

* A description of the vulnerability.
* A proof of concept or steps to reproduce.
* Potential impact.

We will acknowledge your report within 48 hours and provide a timeline for a fix if applicable.
