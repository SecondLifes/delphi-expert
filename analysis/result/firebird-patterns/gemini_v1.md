### [OVERALL] Firebird Patterns Assessment

- Category: OVERALL
- Lens(es): Prompt Engineer Analyst
- Verification verdict: VERIFIED
- Evidence type: static-review-based
- Location: .agents/rules/firebird-patterns.md
- Finding: The rule set provides highly practical, deeply technical guardrails for Firebird development via FireDAC. The warning about using `Open` instead of `ExecSQL` for queries with a `RETURNING` clause is an excellent catch for a notorious Delphi trap. Furthermore, the metadata query section efficiently works around Firebird's lack of `CREATE TABLE IF NOT EXISTS` syntax. However, the code snippets contain multiple Portuguese comments, which introduces language inconsistency.
- Evidence: Code comments such as `{ SEMPRE configurar: }`, `//❌ ExecSQL loses the returned value!` (English), and `{ Tabela existe? }` (Portuguese).
- Impact: Solid behavioral instruction, but code generated by the AI will likely inherit these Portuguese comments, forcing non-Portuguese speaking teams to manually translate or clean up the generated code.
- Recommendation: Unify the language of the code comments to English to match the surrounding instructional text.
